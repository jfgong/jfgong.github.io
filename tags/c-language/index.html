static-extern-const

static静态变量

根据静态变量的作用域, 可以将其分为两种:

  • 静态局部变量, 定义在函数体内, 作用域仅限于该函数
  • 静态全局变量, 定义在函数体外, 作用域仅限于该文件

静态变量虽在程序的整个执行过程中始终存在, 但是在它作用域之外不能使用. 静态变量是对变量作用域的描述, 它不会随着函数的调用和退出而发生变化.它被放在程序的全局数据段中, 上次调用函数的时候, 如果给静态变量赋予某个值, 下次函数调用时，这个值依然保持不变, 这一点是它与堆栈变量和堆变量的区别.

  • 若全局变量仅在单个C文件中访问, 则可将此变量改为静态全局变量, 以降低模块间的耦合度
  • 若全局变量仅由单个函数访问, 则可将此变量改为该函数的静态局部变量, 以降低模块间的耦合度

头文件和原文件

一般来说，函数的声明与定义我们将之分离开来，其中声明放在头文件sample.h中而定义放在对应sample.cpp文件中，所以对于一个函数的调用我们可以归纳为两种情况：

  1. func()声明在sample.h, 定义在sample.cpp

    • 其他cpp调用func()时, include sample.h
    • 其他cpp调用func()时, extern func()

  2. func()定义在sample.cpp, 无对应的头文件

    • 其他cpp调用func()时, extern func()
    • 在other.h中extern func(), 其他cpp调用func()时, include other.h
