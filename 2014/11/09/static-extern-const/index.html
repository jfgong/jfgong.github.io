static-extern-const

static静态变量

根据静态变量的作用域, 可以将其分为两种:

  • 静态局部变量, 定义在函数体内, 作用域仅限于该函数
  • 静态全局变量, 定义在函数体外, 作用域仅限于该文件

静态变量虽在程序的整个执行过程中始终存在, 但是在它作用域之外不能使用. 静态变量是对变量作用域的描述, 它不会随着函数的调用和退出而发生变化.它被放在程序的全局数据段中, 上次调用函数的时候, 如果给静态变量赋予某个值, 下次函数调用时，这个值依然保持不变, 这一点是它与堆栈变量和堆变量的区别.

  • 若全局变量仅在单个C文件中访问, 则可将此变量改为静态全局变量, 以降低模块间的耦合度
  • 若全局变量仅由单个函数访问, 则可将此变量改为该函数的静态局部变量, 以降低模块间的耦合度

静态局部变量

  • 非静态局部变量的作用域只是函数的本次执行, 函数退出后变量被释放, 下次从头再来
  • 静态局部变量的作用域是函数的所有执行, 函数退出后, 变量继续存在, 下次再续前缘

static修饰的全局变量声明与定义同时进行, 也就是说当你在头文件中使用static声明了全局变量后, 它也同时被定义了

静态局部变量定义在函数体内, 并且出现在函数内部的基本类型的静态变量初始化语句只有在第一次调用才执行, 即如果在定义静态局部变量的同时进行初始化, 则再次运行到此处时不再执行初始化操作, 所以强迫症患者不必担心此处存在变量多次定义的问题.

静态局部变量的初始化表达式必须是一个常量或者常量表达式。即使局部静态变量定义时没有赋初值，系统会自动赋初值0(对数值型变量)或空字符(对字符变量), 当多次调用一个函数且要求在调用之间保留某些变量的值时, 可考虑采用静态局部变量.

静态全局变量

  • 非静态全局变量的作用域是整个程序, 该全局变量在程序的所有其他文件中均可使用, extern即可
  • 静态全局变量的作用域只是所在文件, 该全局变量在程序的所有其他文件中不可使用, extern也不行

其实我们可以视静态函数为一种静态全局变量, 该函数遵循同样的规则, 则该函数只在函数定义的文件中有效, 而在其他文件中无法使用.

// func.h
  static int hello = 10;

  void increase();
  void decrease();

// inc.cpp
  #include "func.h"
  #include <stdio.h>

  void increase()
  {
      hello++;
      printf("In increase: hello[%p] = %d\n",&hello, hello);    
  }

// dec.cpp
  # include "func.h"
  # include <stdio.h>

  void decrease()
  {
      hello--;
      printf("In decrease: hello[%p] = %d\n",&hello, hello);    
  }

// main.cpp
  #include "func.h"
  #include <stdio.h>

  int main()
  {
      increase();
      decrease();

      increase();
      decrease();
      printf("In main: hello[%p] = %d\n", &hello, hello);            
      return 0;
  }

上述示例中, 在func.h中定义了静态全局变量, 然后分别在不同的原文件中去包含该头文件, 此处我们只要记住, 包含的头文件即是直接展开在include处, 所以, 该静态变量的定义分别出现在inc.cpp和dec.cpp中, 因为全局静态的作用域仅限于所在文件, 所以他们分别被使用在不同的编译单元中, 虽然名称相同, 但是并不会相互影响, 这也就是static变量的魅力所在. 同时, 在main.cpp中, 也include了func.h, 所以同样存在一个独立的静态变量, 程序输出如下:

In increase: hello[0x601044] = 11
In decrease: hello[0x601048] = 9
In increase: hello[0x601044] = 12
In decrease: hello[0x601048] = 8
In main: hello[0x601040] = 10

可以发现, 如果将全局静态变量定义在头文件中, 其他多个文件包含该头文件时, 会导致存在多个不同的同名静态全局变量, 每一次包含, 都会在相应原文件的作用域中定义一个新的静态全局变量, 这使得模块件耦合度变得很大, 不利于维护. 所以对于静态全局变量, 一般将其定义在原文件而不是头文件中.

const 关键字

首先需要明晰const的概念, 如区分以下几种定义方式:

int a = 500; 
const int* p = &a;        // 常量指针, 不能通过该指针改变内容, 如不能*p=1
int const *p = &a;        // 常量指针
int* const p = &a;        // 指针常量, 不允许改变该指针的指向, 如不能p++
const int* const p = &a;  // 指向常量的常量指针

此处我们可参照另一篇关于指针的文章, 类似于指针数组和数组指针的辨析.

作用于函数参数

const作用于参数时, 通常修饰指针参数和引用参数:

void Fun(const A *in); //修饰指针型传入参数
void Fun(const A &in); //修饰引用型传入参数

虽然使用常量指针后函数不能修改指针所指内容, 但是不代表不能通过其他方式修改, 比如可以通过普通指针去修改, 所以如果不希望函数改变参数的值, 使用const引用不失为一种可行的方法:

// void func(const int &i)
void func(const int *pi)
{
    // *pi = 100; 错误
    int* pp = (int*)pi;
    *pp = 100;
}

const的c++特性

  • const成员函数不可以修改对象的数据, 不管对象是否具有const性质

  • 非const对象的地址可以赋给指向const对象的指针

  • const对象的地址不能赋给普通的非const对象的指针

  • const对象的成员是不能修改的, 所以只能访问const成员函数

  • 非const对象可以访问任意的成员函数, 包括const成员函数

const数据成员只在某个对象生存期内是常量, 而对于整个类而言却是可变的. 因为类可以创建多个对象, 不同的对象其const数据成员的值可以不同.

测试

int a = 0, b = 1;
const int *pa = &a;
int* const pb = &b;

// test code
...

printf("a = %d, ", a);
printf("b = %d, ", b);
printf("*pa = %d, ", *pa);
printf("*pb = %d\n", *pb);

在test code处我们添加以下代码,

Code Output
null a = 0, b = 1, pa = 0, pb = 1
pa = &b a = 0, b = 1, pa = 1, pb = 1
*pa = 1 error: assignment of read-only location ‘* pa’
a = 1; b = 0 a = 1, b = 0, pa = 1, pb = 0
pb = &a error: assignment of read-only variable ‘pb’
*pb = 0 a = 0, b = 0, pa = 0, pb = 0

对于常量来说, 系统没有划定专门的区域来保护其中的数据不能被更改. 也就是说, 使用常量的方式对数据进行保护是通过编译器作语法限制来实现的. 我们仍然可以绕过编译器的限制去修改被定义为“常量”的内存区域.

const int hello = 10;
//int test = 10;
//const int hello = test;
int *p = (int*) &hello;
*p = 11;
printf("hello = %d\n", hello);
printf("*p = %d\n", *p);
printf("%p, %p\n", &hello, p);

实际上, 经过测试, 在gcc version 4.8.2中, 输出为:

hello = 10
*p = 11
0x7fffdcf6c614, 0x7fffdcf6c614

变量hello的值看似并没有被改变, 依然是10, 而*pi的值是11, 那么是不是编译器为p指针重新分配了一块内存呢, 但是从指针的输出来看, hello的地址和指针p的值却是相同的, 那么为什么hello和*p的输出却又不同呢, 我想这应该只能解释为编译器的优化问题了, 编译器应该是在预处理时将代码中的常量hello直接替换成了10, 即printf("hello = %d\n", hello)被改变成printf("hello = %d\n", 10). 如果我们不直接对const hello赋值, 那么编译器将不能进行这种预处理, 如注释行中所示, 此时我们将得到预想的输出:

hello = 11
*p = 11
0x7fff582ab150, 0x7fff582ab150

在c++中, 由于常量对象的状态不允许被修改, 而非常量函数存在修改对象状态的可能性, 所以不能通过常量对象调用非常量成员函数, 同样的, 在常量成员函数中虽然不能修改成员变量的值.

我们知道每个成员函数都有一个隐含的指向对象本身的this指针, 而常量函数则包含一个this的常量指针

void inspect(const Fred* this) const;
void mutate(Fred* this);

只要我们愿意, 还是可以通过常量函数修改对象的状态. 同理, 对于常量对象, 我们也可以构造另外一个指向同一块内存的指针去修改它的状态.

虽然我们可以绕过编译器的错误去修改类的数据成员, 但是C++也允许我们在数据成员的定义前面加上mutable, 以允许该成员可以在常量函数中被修改.

extern关键字

extern可以置于变量或者函数前，以标示变量或者函数的定义在别的文件中，提示编译器遇到此变量和函数时在其他模块中寻找其定义。此外extern也可用来进行链接指定。

extern有两个作用:

  • 设定函数的编译方式, 如extern "C" void func(int a, int b);, 则编译器在编译函数func时按着C的规则去翻译相应的函数名而不是C++的, C++为了支持函数重载, 会加上函数参数, 这样func可能会翻译成func@aBc_int_int#%$

  • 声明函数或全局变量, 如extern int g_Int, 声明的函数和变量可以在本模块使用, 这样可以通过编译阶段, 链接时会从其他模块的目标代码中找到此函数

extern用在变量声明中常常有这样一个作用, 你在.c文件中声明了一个全局的变量, 这个全局的变量如果要被引用，就放在.h中并用extern来声明.

如果函数的声明中带有关键字extern, 仅仅是暗示这个函数可能在别的源文件里定义, 没有其它作用. 即extern int func();和int func();两个函数声明没有明显的区别.

总结

  • extern和static不能同时修饰一个变量
  • static变量的值是可变的, const常量的值不能改变
  • 把局部变量改变为静态变量后是改变了它的存储方式，即改变了它的生存期
  • 把全局变量改变为静态变量后是改变了它的作用域，限制了它的使用范围
