使用hexo在github上搭建主页

前段时间一直用jekyll在github上搭建的主页，但是当文章数量增多，管理越来越麻烦，看到hexo支持tags，并且可以方便的使用目录分类，所以开始转向hexo阵营。

配置环境

安装git，git官网
安装nodejs，node.js官网
安装hexo，hexo官网

以上的环境配置此处不费口舌，可以参考各自的官方网站。

hexo创建主页

进入github，创建一个空repository，按照username.github.io的格式命名

进入任意一个目录，hexo init hexo_blog

cd hexo_blog
npm install
hexo server

至此，已经可以通过localhost:4000来查看新搭建的主页了。

配置hexo

修改配置文件_config.yml

deploy:
  type: github
  repository: git@github.com: username/username.github.com.git
  branch: master

然后执行:

hexo generate
hexo deploy

其中，generate在本地先把博客生成的一套静态站点放到public文件夹中，而deploy会将public复制到.deploy中。成功发布到github后，可以发现对应repository项目中即是目录.deploy下的文件。

此时主页已经可以通过username.github.io来访问，剩下来的工作便是绑定域名，即在文件CNAME中写入域名。由于每次执行deploy的时候，master分支所有的文件都会被覆盖，所以最好在source目录下创建这个CNAME文件，这样每次部署就不用动手创建了。

更新完CNAME并deploy后，CNAME文件便已经push到了项目中，稍微等待一段时间后，就可以使用绑定的域名来访问我们新建的主页了。当然，在此之前你还需要将自己的域名解析到username.github.io，如本主页使用的是DNSPOD提供的域名解析服务。

使用hexo

新建文章并提交

hexo new "postname"
hexo generate
hexo deploy

用户可以随意修改source/_post中的文件名以及文件内容，重新generate&deploy后，github中原文件被删除，只会保留当前最新的内容。

备注：

1.在hexo deploy步骤中，可能会遇到以下错误：

ERROR: Repository not found.
fatal: Could not read from remote repository.
Please make sure you have the correct access rights and the repository exists.

此时，我们可以尝试重新配置文件_config.yml

deploy:
  type: github
  repository: https://github.com/username/username.github.io.git
  branch: master

此外，当deploy提交无效时，可以删除.deploy，然后先generate再deploy

2.在deploy时，可能会出现错误

Permission denied (publickey).
fatal: Could not read from remote repository.

请自行创建key到～/.ssh目录下，然后上传该key文件内容到github中

3.在hexo init步骤中，可能会遇到以下错误：

Error: Theme landscape does not exist.

请检查当前执行目录下是否有_config.yml等文件，有的话删除这些文件，或者尝试更改执行目录。
